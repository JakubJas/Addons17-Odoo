ServiLOPD

Gestión de solicitudes LOPD/GDPR para clientes y proveedores en Odoo

ServiLOPD es un módulo para Odoo 17 diseñado para gestionar el envío, recepción, control y trazabilidad de solicitudes LOPD/GDPR asociadas a clientes y proveedores.

El módulo permite solicitar información actualizada a los contactos, enviarles un formulario seguro mediante enlace único, registrar la aceptación del documento LOPD y actualizar automáticamente la ficha del cliente o proveedor dentro de Odoo.

Funcionamiento general

El funcionamiento de ServiLOPD se basa en un flujo sencillo y controlado:

  1. El usuario selecciona un cliente o proveedor desde Odoo.
  2. Desde la ficha del contacto se pulsa el botón Enviar formulario LOPD.
  3. El sistema crea una solicitud LOPD asociada a ese contacto.
  4. La solicitud genera automáticamente un token único y una fecha de expiración.
  5. El cliente recibe un correo electrónico con un enlace seguro al formulario.
  6. El cliente accede al formulario, revisa sus datos, los modifica si es necesario y acepta el documento LOPD.
  7. Odoo registra la respuesta, actualiza la ficha del contacto y cambia el estado LOPD correspondiente.

Formulario público seguro

Cada solicitud LOPD genera un enlace único asociado a un token. Este enlace permite acceder a un formulario público sin necesidad de que el cliente tenga usuario de Odoo.

El formulario se entrega precargado con los datos ya existentes en la ficha del contacto para que el cliente pueda confirmarlos o corregirlos.

El formulario permite recoger y actualizar, entre otros, los siguientes datos:

  • Nombre de empresa
  • Nombre completo
  • Email
  • CIF, NIF o NIE
  • Teléfono
  • Móvil
  • Calle
  • Código postal
  • Ciudad
  • País
  • Provincia

Los campos principales pueden configurarse como obligatorios para garantizar que la solicitud no se envíe incompleta.

Documento LOPD versionado

El módulo incorpora una sección específica para gestionar documentos LOPD desde Odoo. Desde esta sección el usuario puede subir el documento legal vigente sin necesidad de modificar código.

Cada documento puede tener:

  • Nombre
  • Versión
  • Archivo PDF
  • Estado activo o inactivo

Al generar una nueva solicitud, el sistema vincula automáticamente el documento LOPD activo. De esta forma, cada aceptación queda asociada a una versión concreta del documento legal.

En el formulario público el cliente puede abrir el documento LOPD desde un enlace seguro antes de marcar la casilla de aceptación.

Aceptación LOPD

Para completar el formulario, el cliente debe aceptar expresamente el documento LOPD mediante una casilla obligatoria.

Cuando el cliente envía el formulario, ServiLOPD registra:

  • Fecha y hora de aceptación
  • Dirección IP desde la que se aceptó
  • Versión del documento LOPD aceptado
  • Solicitud asociada
  • Contacto relacionado
  • Datos enviados por el cliente

Esta información permite mantener una trazabilidad clara de cada aceptación realizada.

Estados LOPD

El módulo gestiona estados para controlar el ciclo de vida de cada solicitud y la situación LOPD del contacto.

Las solicitudes pueden encontrarse en estados como:

  • Borrador
  • Enviado
  • Respondido
  • Expirado
  • Cancelado

En la ficha del cliente o proveedor también se muestra el estado LOPD actual, permitiendo identificar rápidamente si la solicitud está pendiente, enviada, firmada o pendiente de revisión.

Envío individual

Desde la ficha de un cliente o proveedor, el usuario puede enviar una solicitud LOPD de forma individual.

Al pulsar el botón de envío, el sistema:

  1. Crea una nueva solicitud LOPD.
  2. Genera un token único.
  3. Calcula la fecha de expiración.
  4. Asocia el documento LOPD activo.
  5. Envía el email al contacto.
  6. Cambia el estado del contacto a enviado.
  7. Registra el evento en el historial del contacto.

Envío masivo

ServiLOPD permite enviar solicitudes LOPD de forma masiva desde la lista de clientes o proveedores.

El usuario puede seleccionar varios contactos mediante los checkboxes de la vista lista y ejecutar la acción de envío masivo.

Para cada contacto seleccionado, el módulo genera una solicitud independiente con su propio token, enlace, fecha de expiración y documento asociado.

Si un contacto no dispone de email, el sistema puede dejar constancia en el historial del contacto y marcarlo para revisión, evitando que el proceso masivo se detenga por completo.

Reenvío de solicitudes

Desde una solicitud LOPD existente, el usuario puede reenviar el formulario al cliente o proveedor.

El reenvío genera un nuevo token, actualiza la fecha de expiración, mantiene la solicitud asociada al contacto y vuelve a enviar el correo electrónico.

Esta función es útil cuando el cliente no ha respondido, el enlace ha expirado o se necesita repetir el envío.

Expiración automática

El módulo incluye una acción programada que revisa periódicamente las solicitudes enviadas.

Si una solicitud supera su fecha de expiración sin haber sido respondida, el sistema cambia automáticamente su estado a expirado y actualiza el estado LOPD del contacto correspondiente.

Esto evita tener solicitudes antiguas abiertas indefinidamente y permite mantener un control operativo claro.

Actualización automática del contacto

Cuando el cliente completa el formulario, los datos recibidos se guardan en la solicitud y se actualiza la ficha del contacto en Odoo.

El módulo puede actualizar datos como nombre, empresa, email, teléfono, dirección, país, provincia y datos fiscales.

En caso de que algún dato requiera revisión manual, el contacto puede quedar marcado en estado pendiente de revisión.

Historial y trazabilidad

ServiLOPD utiliza el historial de Odoo para registrar eventos relevantes en la ficha del contacto.

Algunos eventos que pueden quedar registrados son:

  • Formulario LOPD enviado
  • Formulario reenviado
  • Formulario respondido
  • Solicitud expirada
  • Aceptación recibida desde formulario web
  • Incidencias o datos pendientes de revisión

Esto facilita el seguimiento interno por parte del equipo comercial, administrativo o de cumplimiento.

Smart button en contactos

En la ficha del cliente o proveedor se incorpora un acceso directo a sus solicitudes LOPD.

Este botón permite consultar rápidamente todas las solicitudes asociadas al contacto, su estado, fechas, documento aceptado e información de trazabilidad.

Vista visual de estados

En la vista lista de contactos, el módulo puede aplicar indicadores visuales según el estado LOPD del cliente o proveedor.

Esto permite identificar de forma rápida qué contactos han firmado, qué contactos tienen solicitudes enviadas, cuáles requieren revisión y cuáles no tienen el proceso completado.
